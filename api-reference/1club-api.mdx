## Overview

The 1Club API provides programmatic access to club and membership management: members, bookings, classes, billing, messaging, and integrations. Use it to build custom apps, sync data, or automate workflows.

## Base URLs

- **Production:** `https://api.1club.ai`
- **Local:** `http://localhost:8000`

## Authentication

Most endpoints require a **JWT** in the `Authorization` header:

```bash
Authorization: Bearer <your_jwt_token>
```

Some endpoints (e.g. admin session-based flows) use **session cookies** (`connect.sid`).

Details are in the **1Club API** tab: each endpoint shows required auth and parameters.

## Interactive docs (Swagger UI)

For a live, try-it-now reference:

- **Production:** [https://api.1club.ai/api-docs](https://api.1club.ai/api-docs)
- **Developer portal:** [https://developer.1club.ai](https://developer.1club.ai) (API docs and keys)

## OpenAPI spec

The full OpenAPI 3.0 spec is available as `api-reference/1club-openapi.json` in this repo. Regenerate it from the 1club server with:

```bash
cd server && npm run openapi:export
```

Then copy the generated `openapi.json` to `docs/api-reference/1club-openapi.json` to update the reference in this site.
