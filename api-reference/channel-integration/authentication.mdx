---
title: "Authentication and setup"
description: "Enable the channel integration and create a scoped API key to sync bookings."
---

The channel integration uses the same API key mechanism as the rest of the [Platform API](/api-reference/introduction). Two things must be in place before you can push bookings.

## 1. Enable the integration

The channel integration must be enabled for the organization whose courts you are selling. Enabling it creates the record that links bookings to your channel and sets the channel name used for attribution.

Ask your 1Club contact to enable the padel community integration for the organization, or enable it under **Settings - Integrations** in the admin dashboard.

## 2. Create an API key

API keys are created in the admin dashboard under **Settings - API Tokens**. See [Creating an API token](/api-reference/introduction#creating-an-api-token) for the full flow.

Grant only the scopes the integration needs:

- `courts:read` - list courts and read their availability
- `bookings:read` - read a booking by its external id
- `bookings:write` - create and cancel bookings

<Note>
  `bookings:write` is required to create or cancel bookings. A read-only key
  cannot write bookings, and the legacy broad read grant does not satisfy write
  scopes.
</Note>

Pass the key as a Bearer token on every request:

```
Authorization: Bearer 1club_sk_live_...
```

The key is tied to one organization, so all courts and bookings are scoped to that organization automatically.

## Rate limits and errors

The channel endpoints share the Platform API's limits and error model:

- **100 requests per minute** per organization. A `429` response includes a `Retry-After` header.
- Errors return a JSON body of the shape `{ "error": "...", "message": "...", "code": "..." }`.

See [Rate limiting](/api-reference/introduction#rate-limiting) and [Errors](/api-reference/introduction#errors) for the full reference.
