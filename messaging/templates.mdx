Message templates are the saved subjects and bodies that show up in the **Template** picker when you compose a manual message. They live separately from the system templates that power transactional emails (booking confirmations, receipts, password resets) under **Settings > Communication > Templates**.

The list page is `/messages-templates` in the admin. Use it when you find yourself retyping the same welcome note, class reminder, or promotional blurb every week.

## What a template stores

Each template carries four fields:

- **Name** - shown in the picker and the list.
- **Subject** - used when the template is applied to an email send.
- **Content type** - `html` (rich text) or `text` (plain).
- **Content** - the body, either HTML or plain text.

Templates have a `type` of `message` (manual sends), `transactional` (system emails), or `receipt` (invoice receipts). The compose picker and this UI only show `message` templates. The other two types are managed under **Settings > Communication > Templates**.

The list also shows an **Active / Inactive** chip and the created date. Inactive templates stay in the database but disappear from the compose picker.

## Creating a template

Click **Add template** on the list page. A dialog asks for name, subject, content type, and content. Save and the template becomes available in the compose picker immediately.

To duplicate a template for a variation, hit the **Copy** action on the row. A copy opens in the dialog with `(Copy)` appended to the name; tweak it and save.

## Variables

Write variables as `{{variableName}}`. They are substituted at send time using the recipient's contact record and your organization's settings. For manual `message` templates, the variables resolved by the server are:

| Variable | Description |
| --- | --- |
| `{{contactName}}` | Recipient's full name |
| `{{contactFirstName}}` | Recipient's first name (for greetings) |
| `{{contactEmail}}` | Recipient's email |
| `{{organizationName}}` | Your organization's name |
| `{{organizationEmail}}` | Your organization's contact email |
| `{{organizationPhone}}` | Your organization's phone number |
| `{{clubName}}` | The recipient's club, when known |
| `{{orgBaseUrl}}` | Your organization's base URL |
| `{{orgLogoUrl}}` | Your organization's logo URL |
| `{{senderName}}` | The staff member sending the message |
| `{{subject}}` | The message subject (handy in body copy) |
| `{{messageDate}}` | The send date |

Variables not in this list ship as literal `{{name}}` text in manual sends.

Booking-specific tokens (`{{entityName}}`, `{{bookingDate}}`, `{{bookingTime}}`, `{{accessPin}}`) and membership-specific tokens (`{{planName}}`, `{{renewalPrice}}`, `{{renewUrl}}`, `{{membershipExpiresAt}}`) only resolve inside automations and system templates, where a booking or membership record is bound to the send. For automation-triggered messages, see [Campaigns](/marketing/campaigns).

<Note>
  Legacy templates may use older aliases like `{{userName}}`, `{{userFirstName}}`, `{{memberName}}`, or `{{name}}`. These still resolve to the contact's name for backward compatibility.
</Note>

## Welcome email example

```text
Subject: Welcome to {{organizationName}}!

Hi {{contactFirstName}},

Welcome to {{organizationName}}! We're glad to have you.

If anything comes up, reply here or reach us at {{organizationEmail}}.

Thanks,
The {{organizationName}} team
```

## Using a template

In [compose](/messaging/compose), open the **Template** picker above the subject. Selecting a template fills in both subject and body. You can edit either freely after applying; the saved template is not touched.

The picker only appears for admin and manager roles. Instructor-only inboxes hide it.

## System vs manual templates

If you want to customize the email a member receives when their booking is confirmed, when their membership is about to expire, or when a payment receipt is generated, you are looking for **system templates** under **Settings > Communication > Templates**. Those expose the booking and membership variable sets, plus a customize-vs-override workflow keyed to a stable `systemTemplateKey`.

The Message Templates list described on this page is for the manual sends you fire from the messaging inbox.
