title: Admin users

Admin user management controls who can access your organization's admin panel and what they can do there. You invite a user, assign them one of the built-in roles, and the role decides which areas they see and which actions they can take.

## Where to find it

Go to **Settings → Users**. The page lists every organization user in a grid, with tabs to filter by role: **All**, **Admin**, **Manager**, **Front Desk**, **Instructor**, and **Member**. Each tab shows a count.

The grid columns are:

- **Contact** - name, email, and avatar
- **Role** - the assigned role, plus an **Owner** chip if the user is an owner
- **Status** - the user's account or invitation status
- **Joined** - the date the user joined (or was invited)

## Roles

1Club has five built-in roles. Every user has exactly one. Roles and their permissions are fixed - you assign a role, but you cannot create custom roles or change a role's permissions.

| Role | Access summary |
|------|----------------|
| **Admin** | Full access to every module, including settings and user management. |
| **Manager** | Full management of members, messaging, operations, staff, marketing, analytics, and inventory; read and create billing. No settings access. |
| **Front Desk** | Members, messaging, operations (bookings, check-ins), point of sale, and read access to analytics and inventory. No settings access. |
| **Instructor** | Their own classes, bookings, and check-ins, plus messaging and their own profile. |
| **Member** | No admin access; uses the member portal and public apps. |

For the full permission matrix, role types, and what each action means, see [Roles and permissions](/reference/roles-permissions). In Settings → Users you can also open the **Role permissions** info icon to view the same grid in-app.

### Owner

One or more Admin users can be marked as an **owner**. Owners bypass permission gates and only owners can delete the organization. Owner is a flag on the user, shown as an **Owner** chip in the grid - not a separate role.

## Inviting a user

1. Go to **Settings → Users**.
2. Click **Invite**.
3. Enter the user's details and pick their **role**.
4. Send the invitation.

The user receives an email invitation. Until they accept, they appear in the grid with an invitation status.

<Note>
Inviting someone as an **Instructor** creates (or links) an instructor profile. If the user doesn't have one yet, you'll be prompted to fill in the instructor details before the role is saved.
</Note>

## User and invitation status

- **Active** - the user has accepted and can log in.
- **Invited** / **Pending invite** - an invitation has been created or sent but not yet accepted.
- **Expired** - the invitation link expired before it was accepted.

## Managing existing users

Click a user (or use the row actions menu) to manage them:

- **Edit** - change the user's role. For a user who is also an instructor, this opens the full instructor editor with the role picker.
- **Resend invitation** / **Send invitation** - re-send an invitation that hasn't been accepted yet.
- **Set password** (Admins only) - set a password for another user directly.
- **Remove** / **Delete invitation** - remove an active user from the organization, or delete a pending invitation. This cannot be undone.

You cannot edit, remove, or set a password for your own account from this page.

## Granting an instructor admin-portal access

An instructor can be given access to the admin portal so they can manage their own classes and bookings. This toggle is controlled from the instructor's record and is available to Admins and Managers only. See [Instructor management](/operations/instructors).

## Best practices

- **Assign the least-privileged role** that lets someone do their job. Front Desk covers most front-of-house work without exposing billing settings or analytics.
- **Reserve Admin for the people who run the organization.** Only Admins can change settings and manage other users.
- **Clean up invitations.** Delete invitations that were never accepted so the user list stays accurate.
- **Review access periodically** and remove users who no longer need it.
